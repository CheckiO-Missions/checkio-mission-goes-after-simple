对于这个特殊的解决方案，我们使用寻找字符串中子串的起始索引。可以使用两种字符串方法中的一种：str.find() 或str.index() 。这些方法也适用于多字符子串。让我们比较一下它们的输出结果： 

line = "table"
# find
print(line.find("b"))
print(line.find("le"))
print(line.find("x"))

输出：

2
3
-1        
            
# index
print(line.index("b"))
print(line.index("le"))
print(line.index("x"))

输出：

2
3
ValueError: substring not found

让我们使用第一种方法，它在字符串中没有子串的情况下返回-1 。它使你的解决方案更加简单和简短。 

word.find(...) # returns starting index of substr (if present) or -1

我需要关于find()方法的参数的帮助。

second 字符必须紧跟在first 之后。这意味着，我们要寻找的是子串first + second 。

例如，在字符串"table" 中，"b" 紧跟在"a" 之后 - 它们之间没有任何字符。由于find() 可以接受多个字符的参数，我们可以说，我们正在寻找"table" 字符串中的"a" + "b" -> "ab" 子串。 

word.find(first + second)

该方法返回一个数字（索引或-1 ）。如何从它那里得到bool？

你的函数应该返回True ，如果子串存在于word 。这意味着，起始索引（find 方法的结果）必须不等于-1 （更大，因为它不可能更小）。让我们用这样的方式来比较它们并获得布尔值。 

-1!= word.find(first + second)
# or
-1 < word.find(first + second)

那么，我的最终解决方案应该是什么样子的呢？

你只需要把你的代码包装成函数表达式并返回比较的结果。

下面是整个解决方案：

def goes_after(word: str, first: str, second: str) -> bool:

    return -1 < word.find(first + second)

你也可以用in 运算器来检查存在。 

def goes_after(word: str, first: str, second: str) -> bool:

    return first + second in word

这里使用了另一个方法index() ，在子串不存在的情况下会引发一个错误（ValueError）。 

def goes_after(word: str, first: str, second: str) -> bool:

    try:
        return word.index(first) - word.index(second) == -1
    except:
        return False
